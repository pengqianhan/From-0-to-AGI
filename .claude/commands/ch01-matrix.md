# 第 1 章自我检验：矩阵与线性代数

用户调用了 `/ch01-matrix`，说明他们刚完成第 1 章的学习。你的任务是帮助他们检验自己是否真正理解了，而不是告诉他们答案。

**检验方式**：逐一提问下面的问题，等用户用自己的话回答后，再给出反馈——哪里说对了，哪里还有偏差，以及如何深化理解。不要一次性把所有问题都抛出去。

---

## 检验问题（按难度递进）

**第一关：概念是否清晰**

问用户：
> 不查任何资料，用一句话解释矩阵乘法的"维度规则"——什么情况下两个矩阵可以相乘，结果是什么形状？

如果他们能回答"(m,k) 乘 (k,n) 得 (m,n)，中间的 k 必须相同"，继续下一关。如果不能，回到 `examples/02_matrix_multiply.py` 的计算过程重新理解。

---

**第二关：直觉是否建立**

问用户：
> 神经网络里有一行代码：`y = X @ W + b`，其中 X 是 (32, 128)，W 是 (128, 64)，b 是 (64,)。不用计算，直接说出 y 的形状，并解释 32、128、64 分别代表什么。

如果他们能回答"y 是 (32, 64)，32 是 batch size，128 是输入特征维度，64 是输出特征维度"，继续下一关。

---

**第三关：能否发现问题**

给用户看这段代码，问他们哪里有 bug：

```python
X = np.random.randn(4, 3)
W = np.random.randn(3, 5)
b = np.zeros(3)
y = X @ W + b
```

正确答案：`b` 的形状应该是 `(5,)` 而不是 `(3,)`，否则广播规则会报错或产生错误结果。

---

**第四关：能否迁移**

问用户：
> 一张 RGB 图片，大小是 32×32 像素，展平后是一个多少维的向量？如果我想用一个线性层把它变成 10 个类别的分数，权重矩阵 W 应该是什么形状？

正确答案：32×32×3 = 3072 维；W 是 (3072, 10)。

---

## 反馈原则

- 如果用户答对了：认可，然后追问一个更深的"为什么"
- 如果用户答错了：不要直接给答案，给一个提示，让他们重新思考
- 如果用户说"我不知道"：让他们先猜一个，哪怕猜错了也要先猜

检验结束后，如果用户通过了所有四关，告诉他们可以进入第 2 章（`/ch02-calculus`）。
